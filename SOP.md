Some items to include:
- Where software repositories will exist
- Software version management
  - How often reviewed
  - What steps are required for review of new libraries [Note: I think is about compliers and 3rd party (Python, numpy, matplotlib, etc.)]
  - What items need to be reviewed by supervisor
- What are the required steps/sequence of the software additions and modifications
  - Specifically include testing and how the testing environment needs to be configured
  - One process is OK. It is also OK to have multiple processed for the type of addition/modification. For example, an update which only changes a report (doesn't change data in the database) may have less stringent requirements compared to an update that modifies information in the database.
